# Campaign Mode v2.0: Image-Based Terrain Generation

## Overview
This feature introduces a dynamic way to generate hex grid terrains for Campaign mode by analyzing a background "Rectangle Map Image" (RMI). Each hex cell's terrain will be determined by the dominant color of the corresponding region on the RMI, using colors inspired by Ragnarok Online (RO) maps.

## Core Concepts

### 1. Hex to RMI Mapping
The hex grid's visual coordinates `(visualX, visualY)` will be mapped to normalized `[0, 1]` coordinates relative to the grid's bounding box, and then scaled to the RMI's pixel dimensions.

- **Grid Bounding Box:** `minX, maxX, minY, maxY` (calculated in `generateHexGrid`).
- **Normalization:**
    - `u = (hex.visualX - minX) / (maxX - minX)`
    - `v = (hex.visualY - minY) / (maxY - minY)`
- **Pixel Coordinates:**
    - `pixelX = u * imageWidth`
    - `pixelY = v * imageHeight`

### 2. Color Sampling
For each hex, we will sample a region around its center on the RMI.
- **Method:** Mean or Median color of a 5x5 pixel area.
- **Benefit:** Smooths out noise from textures or small map details.

### 3. Terrain Classification
The sampled mean color is compared against reference colors using **Euclidean distance** in RGB space:
`distance = sqrt((r1-r2)^2 + (g1-g2)^2 + (b1-b2)^2)`

**Reference Terrain Colors (RO-inspired):**
| Terrain | Reference Color (RGB) | Description |
| :--- | :--- | :--- |
| **PLAIN** | `[145, 198, 77]` | Grasslands (e.g., Prontera Fields) |
| **FOREST** | `[34, 139, 34]` | Dense Woods (e.g., Payon Forest) |
| **LAKE** | `[65, 105, 225]` | Deep Water (e.g., Izlude/Byalan) |
| **TOWER** | `[128, 0, 128]` | Structures/Artificial (e.g., Geffen Tower) |
| **MOUNTAIN**| `[139, 69, 19]` | High Ground/Arid (e.g., Morocc Desert/Mt. Mjolnir) |

## Implementation Details

### A. Level Configuration
Update `level.json` to support RMI:
```json
{
  "name": "The Road to Prontera",
  "rmi": "/assets/ro_maps/prt_fild08.gif",
  "radius": 5,
  ...
}
```

### B. Runtime Generation (game.js)
1.  **Image Loading:** `Image.decode()` to ensure the image is ready for canvas processing.
2.  **Terrain Analysis:**
    - Load RMI into an offscreen `OffscreenCanvas` or temporary `canvas`.
    - Iterate through `this.hexes`.
    - For each hex, sample the RMI at `(pixelX, pixelY)`.
    - Find the terrain with the minimum Euclidean distance to the sample.
    - Assign `hex.terrainType`.
3.  **UI Update:** Set the `background-image` of `.hex-grid-container` to the RMI.

## Visual Enhancements
- **Transparency:** Hexagons should have a semi-transparent background (e.g., `rgba(..., 0.4)`) to let the RMI show through while still indicating the terrain type.
- **Grid Alignment:** Use `background-size: cover` or `contain` and ensure the hex grid normalization matches this scaling logic.

## Future Considerations
- **Terrain Weights:** Adjust the distance calculation to favor certain terrains (e.g., making MOUNTAIN harder to trigger).
- **Custom Color Palettes:** Different RO maps might need different reference colors (e.g., Ice Dungeon needs a "Snow" terrain).
