# Plan: Hex Dice v1.2 Extended Terrain Implementation

## Objective:
Implement v1.2 Extended Terrain rules including R=8 map, terrain generation algorithms, and terrain-specific movement/combat modifiers.

## User Feedback Incorporated:
*   **Terrain Generation UI:** Instant Setup (terrain just "appears," no animation/visibility in log).
*   **Terrain Roulette Placement:** Auto-Placement (simple, not turn-by-turn UI).
*   **Dice 6 Mountain:** Dice 6 cannot enter Mountain hexes.

## Key Files:
*   `game.js`
*   `index.html`

## Verification:
*   New test scenarios for terrain-specific range bonuses, defensive bonuses, and LoS blocking.
*   Manual verification of terrain generation steps.

## Strategy and Implementation Steps:

### Phase 1: Core Architecture Changes

#### Task 1.1: Dynamic Map Scaling (R=8)
*   **Instruction:** Modify `generateHexGrid` and `determineBaseLocations` in `game.js` to support an `R=8` grid size. This involves adjusting loop bounds and coordinate calculations.
*   **File:** `game.js`

#### Task 1.2: Terrain Data Model
*   **Instruction:** Add a `terrainType` property to the hex object within `generateHexGrid` in `game.js`, initializing it to a default (e.g., 'PLAINS' or `null`).
*   **File:** `game.js`

#### Task 1.3: UI/Visuals Adaptation
*   **Instruction:** Update `hexColor` function (or equivalent rendering logic) in `game.js` to visually represent different `terrainType`s (e.g., 'FOREST', 'LAKE', 'TOWER', 'MOUNTAIN').
*   **Instruction:** Adjust CSS (likely in `index.html` or a linked stylesheet) to ensure the larger R=8 grid fits the container without overflow or visual distortion.
*   **Files:** `game.js`, `index.html` (or `litewind.css` if applicable).

### Phase 2: Terrain Setup Logic

#### Task 2.1: `setupTerrain` Method
*   **Instruction:** Implement a new function `setupTerrain` in `game.js` that will be responsible for orchestrating terrain generation based on the game mode (Roulette or 4-roll Generation).
*   **File:** `game.js`

#### Task 2.2: Terrain Generation - Roulette Mode (Auto-Placement)
*   **Instruction:** Within `setupTerrain`, implement the "sum-of-6-dice and clockwise placement logic" for Roulette mode. This should auto-place terrain types based on pre-defined rules, without user interaction.
*   **File:** `game.js`

#### Task 2.3: Terrain Generation - 4-Roll Algorithm (R=8)
*   **Instruction:** Within `setupTerrain`, implement the "4-roll algorithm (Direction, Distance, Scatter, Terrain Type)" for the R=8 grid. This will procedurally generate terrain.
*   **File:** `game.js`

### Phase 3: Terrain Rules Implementation

#### Task 3.1: Movement - Impassable & Cost Modifiers
*   **Instruction:** Update `bfsValidMoves` (or equivalent movement validation logic) in `game.js`:
    *   Mark `LAKE` hexes as impassable for all dice.
    *   Mark `MOUNTAIN` hexes as impassable for all dice (based on user feedback for Dice 6).
    *   (Original plan mentioned reduced range for some dice on mountain, but user feedback made mountain impassable. Re-evaluate if any other movement cost logic needs to be handled.)
*   **File:** `game.js`

#### Task 3.2: Combat & Defense Bonuses
*   **Instruction:** Update `calcDefenderEffectiveArmor` (or equivalent combat calculation) in `game.js` to include defensive bonuses:
    *   `FOREST`: +1 armor
    *   `TOWER`: +1 armor
    *   `MOUNTAIN`: +2 armor
*   **File:** `game.js`

#### Task 3.3: Ranged Attacks (Dice 5) - Range & LoS
*   **Instruction:** Update `calcValidRangedTargets` in `game.js` for Dice 5 ranged attacks:
    *   `TOWER`: Range 1-3
    *   `MOUNTAIN`: Range 2-4
*   **Instruction:** Update `hasLineOfSight` in `game.js`:
    *   Block LoS for `FOREST`, `TOWER`, and `MOUNTAIN`.
    *   `LAKE` remains transparent for LoS.
*   **File:** `game.js`

### Phase 4: Initialization & Activation

#### Task 4.1: URL Parameter Detection
*   **Instruction:** In `game.js` (or `index.html` if simple script check), implement logic to detect `?v=1.2` in the URL query parameters.
*   **File:** `game.js`

#### Task 4.2: Game Initialization with v1.2 Rules
*   **Instruction:** If `?v=1.2` is detected, modify the game initialization sequence in `game.js` to:
    *   Default the grid size to `R=8`.
    *   Call the newly implemented `setupTerrain` function before game deployment.
*   **File:** `game.js`

Begin implementing Phase 1, starting with dynamic map scaling and terrain data model.
